# 📊 Automated Hosting Guide for HTML 3D Charts

This document provides specialized instructions for hosting your **static HTML visualization reports** on GitHub Pages. Note that this process only handles the deployment of chart artifacts for online viewing and does not involve deploying the project source code or any backend services.

---

## 🚀 Visualization Hosting Workflow

### 1. Fork the Repository
To host charts under your own GitHub domain, start by **Forking** this repository to your personal account.

### 2. Enable GitHub Pages for Reports (One-time Setup)
Once the repository is forked, configure the hosting settings on GitHub:
1.  Navigate to your fork's **Settings** tab.
2.  Select **Pages** from the left-hand menu.
3.  Under **Build and deployment > Source**, ensure `Deploy from a branch` is selected.
4.  Under **Branch**, select `gh-pages` and set the folder to `/(root)`.
    *   *Note: The `gh-pages` branch will be automatically generated by the script during its first successful run.*
5.  Click **Save**.

### 3. Application Configuration
Ensure that chart generation and deployment are enabled in your active YAML configuration (e.g., `configs/default.yaml`):

```yaml
visualize:
  enabled: true   # Required to generate the HTML files
  
deploy:
  enabled: true   # Required to push charts to the web branch
  branch: "gh-pages"
```

### 4. Executing the Pipeline
Run the scan pipeline to generate and host your charts:

```bash
python gigaalpha/scripts/scan.py --config configs/default.yaml
```

The system will automatically:
- Generate interactive 3D HTML charts.
- Synchronize these files with your `gh-pages` branch.
- Push the updates to your GitHub mirror.

### 5. Viewing Your Charts Online
Your interactive reports will be available at:
`https://<YOUR_GITHUB_USERNAME>.github.io/GigaAlpha/outputs/html/<REPORT_NAME>.html`

---

## 🛠 Troubleshooting

- **Access Denied**: Verify that your local environment has authenticated push access to your GitHub account (via SSH or Personal Access Token).
- **Hosting Delay**: It typically takes 60-120 seconds for GitHub to refresh the live site after a new deployment.
- **Scope Limitation**: This deployment is strictly for static `.html` artifacts. It does not support dynamic server-side logic or database connections.
